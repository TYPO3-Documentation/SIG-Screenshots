Screen configuration¶

The screen configuration is an object with the following parameters:

view¶
Required:true
Type:string

Must contain one of the following:

table
Take a screenshot of the table overview in list view on a certain page. This page is defined by pid or selectRecord configuration.
record
Take an overview screenshot of one record. The record is defined by uid or selectRecord configuration. Tabs can be switched by actions.
fields
Take screenshots of (multiple) fields as defined in the property fields of the corresponding table configuration. The record is defined by uid or selectRecord configuration. Tabs can be switched by actions.
pid¶
Required:false
Type:integer

Defines the uid of the page on which a screenshot of the table overview should be taken. Only used in view table, overrides a selectRecord configuration.

uid¶
Required:false
Type:integer

Uid of the record whose screenshot should be taken. Only used in view record and fields, overrides a selectRecord configuration.

selectRecord¶
Required:false
Type:select configuration

Defines the uid of the page on which the screenshots should be taken (view table) or the record (view record and fields).

Example

"selectRecord": {
   "by": "header",
   "header": "Playground"
}
actions¶
Required:false
Type:array of action configurations

The ref:actions <action_configuration> that should be executed before taking the screenshot. They include clicking an element, waiting for a defined time or selector, changing tabs, etc. The actions will be executed in the order as defined in this array.

In the view fields it can be overridden by the actions defined in the field.

selector¶
Required:false
Type:string

Selector to choose the div of which a screenshot should be taken. If empty a screenshot of the whole page will be taken. Example: .recordlist.

prefix¶
Required:false
Type:string (in lower_case_with_underlines)

A prefix used for all filenames and links. For example inline_mn_. Should end on a underline

filename¶
Required:false
Type:string

The desired filename without extensions. For example CtrlType. In the view fields it is overridden by the filename defined in the field.

Defaults to the table name in UpperCamelCase for views record and table.

Must be unique within the defined path. Otherwise both screenshots and corresponding ReST files will be silently overridden.

caption¶
Required:false
Type:string

A caption to be displayed below the screenshot. In the view fields it is overridden by the caption defined in the field.

screenshot¶
Required:false
Type:string
ignore
Don’t attempt to take a screenshot. This can be used to comment out a screenshot that isn’t working right now or to only generate the code snippet.
ignoreOnError
Try to take the screenshot but ignore an error. Can be used in development to ignore screenshots that temporary fail. Shouldn’t be used in production.
default
Take the screenshot.
snippet¶
Required:false
Type:string
ignore
Don’t attempt to create a code snippet.
default
Try to create a code snippet if possible.
